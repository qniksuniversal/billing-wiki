## Glossary of terms

This page shows explanations of the terms that we use on our hosting. In other contexts, these meanings may mean something else; this is just an explanation for this or that word as we understand it and work with it.

### Server

your service in the panel, we are not used to using the word Service in those. support, etc. That's why we call all your services servers.

### Dedik

dedicated server (from English - dedicated server). From the word dedicated. We call it our server where you install your services, for example your website or game server - they all stand on a large server where there are no virtual resources. That is, we control the power supply, manual settings and manually reinstall the OS if necessary.

### Wheelbarrow

This is a synonym for the word dedik.

### Ticket

from English ticket. Translated as a ticket. This indicates your support request. It took a long time for clients, like us, to say “Check my request to technical support”, but instead it was simply “Check the ticket”.
Logically, this literally means a ticket - a ticket to support, like a pass.

### Log(s)

from the English word logs. This means magazines. People from the CIS sometimes also simply say log, as in 1 piece. This means something completely different in English - a log. Developments in those same states don't say that. Log(s) is information that the server stores as a result of its actions. For example, you are starting a server, how do you know that it has started? The server returns the log and says, for example, [POWER] Server is running. Or for example, how to understand what the error is? You need to read the logs, for example the application could not start - Out of memory: true. We - developers, can simply translate this and find out "Running out of RAM: yes", which means we know what happened - out of RAM.

### Bug

from English bug. It literally means error. For example, you click on a button - it should register you, but for some reason it spends money from your card. This is called a bug.

### Debug(-live)

this is a process or action when a developer wants to find why this or that bug happened. It comes from the English detect bug. Abbreviated as debug. Sometimes you can hear - I'm debugging the script, this means that the admin checks the logs and results of the work in order to find why the bug appeared and how it works.

### Script

it literally translates as a script. If, for example, many programs work on a large number of files or code, then a script is one or just a few files that are needed to find a bug or other actions. For example, I run a script to check the weather forecast - this means that most likely it is 1 file in which an api request is sent to the weather service.

### API

this is a group that denotes methods for sending and receiving data. For example: you want to make a button on the site that will log you into your account, the site sends an API request to the backend. Or the API can, on the contrary, return something, for example, you yourself make a request to the API - and it returns to you how the weather is today. This is necessary to connect your application with a database or server that processes data. There are many API requests, the most popular ones. PUT, GET.
PUT - send (and receive a response), GET - receive (most often).

### Backend and frontend

backend is a word taken from the English backend, those who have studied English know that back is behind. People thought logically and decided that the backend is literally the back part of the application. In reality, this means what you send requests to (for example, ordering pizza or registering). The backend is like a connection between the client interface (correct output of all information) and the response made for each button, as well as a connection with the database where it saves (if necessary) the received information. Frontend - from the English frontend. Front from English is in front, it is logical that frontend is the front part. That is, this is the interface, beautiful graphics and other information that a person sees. That is, the entire Wikipedia that you are reading right now is front-end. The backend is, for example, a server that processes your requests for server renewal, and the front just displays beautifully.

### TK

Technical specifications are usually created before the project. For example, I want a discord bot for auto-moderation that will work in Python. You give it to the developer, he makes a project for you.

### Ishews

this is a synonym for the word technical task, only if technical task is to do something before the release of the project (most often), then idea is a task for the developer what exactly and where to fix it (with more specific edits and clarifications).

### Backup

backup copy, from English backup or go back. They are needed to restore to the previous save point in the event of a breakdown.

### DB or DB

from the word "database". It literally means the same thing. Why does DB sometimes occur? DB from the word database, that is, it means the same thing, it’s just written differently in different languages.

### IP address and Port

These are numbers in a specific context. For example, you see 255.0.0.1:25565, here the port will be 25565, and the IP address will be 255.0.0.1.

### Domain

this is the literal name for your server. For example google.com. All domains on the Internet are paid; among the free ones, you can use .pp.ua or our subdomain. The domain is connected to one or more IP addresses, for example google.com -> 1.1.1.1. This is called A record binding. If a domain points to another domain, for example google.com -> c1.kernix.cloud, this is called a CNAME. Important! When one domain redirects to another, this does not mean that when you go to google.com you will be taken to another domain - no! This means that it is easier for the provider or hosting to connect yours via a letter IP. This is convenient because if the provider moves to a new IP, he changes the IP, but you don’t. This makes it less fuss.

### Sub domain

A subdomain is a prefix in front of a domain. For example play.google.com. If we are talking about google.com, then this is a domain, and play.google.com is already a subdomain.

### Acquiring

payment service through which you make payments. For example, we use mono acquiring, when replenishing your balance - you replenish using it.
